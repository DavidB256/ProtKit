ProtKit v1.0

By David Bass

Welcome to ProtKit!

ProtKit is a suite of bioinformatics tools for working with .pdb files created by David Bass for the 2021 Virginia iGEM team. ProtKit is written in C++17 and has support for Windows.

Why ProtKit?

ProtKit was created as an open-source tool to grant structural biologists greater control over the files that they work with. It brings a number of utilities together into one user-friendly, command-line program.

Running ProtKit

ProtKit can downloaded as a .exe file from the "Releases" page of this repository. ProtKit.exe should be stored in the same folder as a file titled "config.txt" that stores the path of the working directory and the default file name used by the "load_working" commands. See the example config.txt for syntax. If no config file is present, ProtKit will default to "C:/Users/Public/" for the working directory and "output.pdb" for the default file name.

When opened, ProtKit will appear as a black command-line window. Commands can be typed directly into the window upon startup.

Loading files

The first thing to do with ProtKit is to load in .pdb files from your working directory. Use to place a file at the end of the loaded files vector. Upon loading a file, it will become "active." The active file can be changed to previously-loaded files in the loaded files vector with the forward, back, and find commands. The working file, which will be discussed in the next section, can be added to the end of the loaded files vector with the load_working command.

Working with files

The majority of ProtKit's commands exist to manipulate loaded .pdb files.

Running scripts

All ProtKit commands can also be run from pre-written scripts. To use a script, type commands (with one command per line) into a .txt file. The script file can then be renamed with any extension, as can be seen with our use of the .t extension in our examples.

Viewing files

To view .pdb files produced with ProtKit, we recommend using NGL Viewer's web app or PyMOL.
