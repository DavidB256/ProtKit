Return to home page.

quit

Quits the current instance of ProtKit, closing the console and erasing all instance data.

Abbreviated name: q

reset

Resets current instance by deleting, then reinitializing, all variables.

Abbreviated name: re

reset_working

Resets working file by deleting, then reinitializing it.

Abbreviated name: rw

set_wd <path>

Sets the current working directory, from which files are loaded and to which files are be written, to the given "path" argument. Recall that the working directory can be set in config.txt and defaults to "C:\\Users\\Public\\".

script <input file>

Runs commands written in file with name "input file". See scri

Abbreviated name: s

load <input file>

Loads "ATOM" lines from .pdb file with name "input file". If no file extension is given, it will add ".pdb" to the end of "input file".

load_het <input file>

Loads "HETATM" lines from .pdb file with name "input file". If no file extension is given, it will add ".pdb" to the end of "input file".

print_AA

Prints the amino acid sequence of the active file followed by its length.

print_loaded

Prints the names of all loaded files, in order, and indicates which file is active.

write_fasta <output file>

Writes .fasta file containing name and amino acid sequence of active file. For details on .fasta files, see the Zhang Lab's "What is FASTA format?" page.

back

Sets the active file to the file loaded before the current active file.

forward

Sets the active file to the file loaded after the current active file.

find <file>

Sets the active file to the first loaded file with name "file", if there is one.

del_active

Removes active file from list of loaded files.

print_active

Prints name of active file.

print_3_letter_AA_seq

Prints amino acid sequence of active file, but with single-letter codes translated into respective space-separated three-letter codes.

write <output file>

Writes working file into .pdb file with name "output file". If "output file" does not have a file extension, it is automatically assigned the ".pdb" extension.

Abbreviated name: w

self

Writes active file into working file. Equivalent to "add_vector 0 0 0" or "mult_matrix 1 0 0 / 0 1 0 / 0 0 1".

add_vector <translation 3-vector>

Writes active file into working file, but with all atoms translated by adding "translation 3-vector" to their coordinates. "translation 3-vector" should be given as 3 space-separated numbers, e.g. "add_vector x y z".

mult_matrix <rotation 3x3 matrix>

Writes active file into working file, but with all atoms transformed by left-multiplying their coordinates by "rotation 3x3 matrix". "rotation 3x3 matrix" should be given as 3 lines of 3 space-separated numbers, separated with " / ", e.g. "mult_matrix x1 y1 z1 / x2 y2 z2 / x3 y3 z3".

axis_rotate <axis 3-vector, point 3-vector, angle>

Writes active file into working file, but with all atoms transformed by rotating "angle" radians around the axis parallel to "axis 3-vector" that goes through the point "point 3-vector". The three arguments should be separated with " / ", e.g. "axis_rotate x1 y1 z1 / x2 y2 z2 / a".

move_to <destination 3-vector>

Writes active file into working file, but with its atoms translated such that the mean position of all of its atoms is at "destination 3-vector". "destination 3-vector" should be given as 3 space-separated numbers, e.g. "move_to x y z".

center

Writes active file into working file, but with its atoms translated such that the mean position of all of its atoms is at the origin. Equivlanet to "move_to 0 0 0".

print_rmsd

Prints root-mean-square deviation (RMSD) between the aligned alpha-carbons of the active file and the file loaded before the active file, in Angstroms. Greater RMSD values indicate that the two proteins differ more in their structures. For details on RMSD calculation and interpretation, see Wikipedia's

print_tm_score

Prints template modeling score (TM-score) between the aligned alpha-carbons of the active file and the file loaded before the active file, in Angstroms, using the prior file as the target. Template modeling score takes on values between 0 and 1, with greater values indicating greater structural similarity. Note that TM-score is generally considered to be a better alternative to RMSD because it is less susceptible being skewed by large single-bond angle deviations. For details on TM-score calculation and interpretation, see Wikipedia's "Template modeling score" page.

load_working

Loads the working file into the list of loaded files, giving it a placeholder name. Used for working with modified or combined protein structures without having to circuitously use "write " followed by "load ".

print_bounds

Prints the coordinates of the points of opposite corners of the minimal rectangular prism that bounds the protein in the active file.
